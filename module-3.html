第3章：首次项目创建

创建新项目流程、项目结构和基本操作指南

学习目标

  • 掌握在KIRO IDE中创建新项目的完整流程
  • 理解KIRO项目的标准结构
  • 学会使用KIRO的规范（Spec）驱动开发方法
  • 熟悉基本的项目管理操作
  • 了解项目配置文件的作用

项目创建概述

在KIRO IDE中创建项目与传统IDE有所不同。KIRO采用规范驱动开发（Spec-driven Development）的方法，这意味着项目创建不仅仅是设置文件结构，还包括定义项目的功能规范。这有助于AI助手更好地理解您的项目需求并提供更精准的帮助。

创建新项目步骤

以下是创建新项目的详细步骤：

启动KIRO IDE

首先确保KIRO IDE已经正确安装并启动。如果这是您首次使用KIRO，可能需要完成登录和初始配置步骤。

打开项目创建界面

在KIRO IDE的启动界面，点击 "Create New Project" 按钮，或者在菜单栏选择 "File" > "New Project"。

选择项目模板

KIRO提供多种项目模板来帮助您快速开始：

  • Basic Project：基本的项目结构，适合简单应用
  • Web Application：包含前端和后端的Web应用模板
  • API Service：REST API服务模板
  • Mobile App：移动应用开发模板
  • Custom：自定义项目结构

对于首次项目，建议选择 "Basic Project"。

配置项目信息

在项目配置界面中，填写以下信息：

  • 项目名称：输入项目的名称（例如 "HelloKiro"）
  • 项目位置：选择项目在本地的存储路径
  • 项目类型：选择项目的主要编程语言
  • 版本控制：选择是否初始化Git仓库

定义初始规范（Spec）

这是KIRO项目创建的核心步骤。在项目创建界面中，您需要定义项目的初始规范：

# HelloKiro 项目规范 ## 项目概述 创建一个简单的Hello World应用，展示KIRO IDE的基本功能。 ## 功能需求 1. 在控制台输出 "Hello, KIRO IDE!" 2. 接受用户输入并个性化问候 3. 显示当前时间 ## 技术要求 - 使用JavaScript语言 - 代码应具有适当的注释 - 遵循基本的编码规范

完成项目创建

点击 "Create Project" 按钮，KIRO IDE将根据您定义的规范生成项目结构和初始代码。

KIRO项目结构

创建项目后，您将看到以下标准项目结构：

.kiro-project/ ├── .kiro/ │ ├── config.json # KIRO项目配置 │ ├── specs/ # 规范文件目录 │ │ └── main.spec # 主规范文件 │ └── workspace.json # 工作区配置 ├── src/ # 源代码目录 │ ├── main.js # 主程序文件 │ └── utils.js # 工具函数 ├── tests/ # 测试文件目录 ├── docs/ # 项目文档 ├── README.md # 项目说明文件 └── package.json # 项目依赖配置（如适用）

关键目录和文件说明：

  • .kiro/：KIRO专用配置目录，包含项目配置和规范文件
  • specs/：存储项目规范文件，这是KIRO的核心特性之一
  • src/：源代码目录，存放主要的程序代码
  • tests/：测试代码目录
  • docs/：项目文档目录

规范（Spec）文件详解

规范文件是KIRO IDE的核心概念，它使用特定的格式来描述项目需求：

# 项目规范示例 ## 模块概述 描述该模块的功能和目的 ## 功能需求 - 功能点1：详细描述 - 功能点2：详细描述 - 功能点3：详细描述 ## 输入输出 - 输入：数据类型和格式要求 - 输出：期望的输出格式 ## 边界条件 - 特殊情况处理 - 错误处理要求 ## 性能要求 - 响应时间要求 - 资源使用限制

当您修改规范文件时，KIRO的AI助手会分析变更并建议相应的代码修改，这大大提高了开发效率和代码质量。

基本操作指南

创建项目后，您可以执行以下基本操作：

1. 编辑代码

在 src/ 目录中打开主程序文件，您可以开始编辑代码。KIRO会根据规范文件提供智能代码补全和建议。

2. 运行项目

在KIRO IDE中运行项目非常简单：

  1. 确保主程序文件处于活动状态
  2. 点击工具栏中的 "Run" 按钮，或使用快捷键 Ctrl+R (Windows/Linux) 或 Cmd+R (macOS)
  3. 观察终端输出以查看程序运行结果

3. 查看AI建议

KIRO的AI助手会持续分析您的代码和规范，提供改进建议：

  1. 在代码编辑器中，您会看到AI建议的提示标记
  2. 将鼠标悬停在提示标记上查看建议详情
  3. 点击建议可以选择接受或拒绝

4. 更新规范

随着项目开发的进行，您可能需要更新规范文件：

  1. 在 .kiro/specs/ 目录中打开规范文件
  2. 添加新的功能需求或修改现有需求
  3. KIRO会分析变更并建议相应的代码修改

项目配置

KIRO项目包含几个重要的配置文件：

.kiro/config.json

这个文件包含项目的KIRO特定配置：

{ "projectName": "HelloKiro", "language": "javascript", "aiModel": "default", "codeStyle": "standard", "autoSave": true, "aiSuggestions": true }

package.json（对于JavaScript项目）

如果您的项目是JavaScript项目，会包含标准的package.json文件来管理依赖。

小贴士

在KIRO中，规范文件不仅仅是文档，而是开发过程的驱动因素。维护清晰、详细的规范将使AI助手能够提供更准确的帮助，从而提高开发效率。

常见问题及解决方案

问题1：项目创建失败

原因：可能是路径权限问题或路径包含特殊字符

解决方案：确保选择的项目路径有适当的读写权限，避免使用包含空格或特殊字符的路径。

问题2：AI助手没有响应

原因：网络连接问题或AI服务不可用

解决方案：检查网络连接，确保有稳定的互联网连接，然后重新启动KIRO IDE。

问题3：规范文件格式错误

原因：规范文件语法不正确

解决方案：确保使用正确的Markdown语法，KIRO规范文件本质上是特殊的Markdown文件。

最佳实践

以下是使用KIRO创建项目时的最佳实践：

  • 详细定义规范：在项目开始时尽可能详细地定义规范，这将使AI助手能够提供更准确的帮助
  • 定期更新规范：随着项目的发展，及时更新规范文件以反映新的需求和变更
  • 使用版本控制：始终启用Git版本控制来跟踪代码和规范的变更
  • 模块化开发：将复杂功能分解为多个小模块，每个模块可以有自己的规范文件
  • 定期审查AI建议：虽然AI建议通常很有帮助，但仍需人工审查以确保符合项目需求
