Autopilot功能概述

Autopilot是KIRO IDE中的一项强大功能，它允许AI助手自动执行一系列开发任务，从代码生成到测试运行，大大提高开发效率。

什么是Autopilot

Autopilot是一种半自动化的开发模式，它结合了AI的智能决策能力和开发者的指导，能够自动完成复杂的多步骤任务。与传统的交互式聊天不同，Autopilot可以在较少人工干预的情况下完成整个开发流程。

Autopilot的工作原理

Autopilot通过以下步骤工作：

  1. 任务理解：AI分析您的需求并确定执行步骤
  2. 计划制定：AI创建详细的执行计划
  3. 执行确认：AI展示计划并请求您的确认
  4. 自动执行：获得确认后，AI逐步执行计划中的任务
  5. 结果反馈：AI报告执行结果并请求进一步指示

启用Autopilot

要启用Autopilot功能，请按照以下步骤操作：

  1. 在Chat界面中，点击右上角的设置图标
  2. 在弹出的菜单中，选择"Autopilot模式"
  3. 确认启用后，Chat界面将显示Autopilot特有的UI元素

小贴士

Autopilot最适合处理具有明确开始和结束点的任务，如创建新功能、重构代码或修复bug。对于探索性任务，传统的交互式聊天可能更合适。

Autopilot使用示例

以下是一个使用Autopilot创建新功能的示例：

用户：我想为我的React应用添加一个用户登录功能，包括登录表单、验证和状态管理。 AI：我将为您创建一个完整的用户登录功能。计划如下： 1. 创建登录组件(LoginForm.js) 2. 添加表单验证逻辑 3. 实现状态管理(useContext) 4. 添加路由保护 5. 创建登录API接口模拟 是否继续执行此计划？[是/否/修改计划]

Vibe vs. Spec模式

KIRO的Chat功能提供两种主要交互模式：Vibe模式和Spec模式。这两种模式适用于不同的开发场景和需求。

Vibe模式

Vibe模式是一种更自由、更灵活的交互方式，适合以下场景：

  • 创意探索：当您不确定具体实现方案时
  • 概念验证：快速验证想法的可行性
  • 学习新技术：通过对话了解新概念和最佳实践
  • 头脑风暴：与AI协作产生新想法

在Vibe模式下，AI会更倾向于提供多种选项、解释概念，并引导您思考不同的解决方案。

Spec模式

Spec模式是一种更结构化、更精确的交互方式，适合以下场景：

  • 具体实现：当您已有明确的需求和规范时
  • 代码生成：需要生成符合特定标准的代码
  • 重构任务：需要精确修改现有代码结构
  • 测试编写：需要生成符合特定测试框架的测试代码

在Spec模式下，AI会更专注于提供精确、可执行的代码和指令，减少解释性内容。

如何切换模式

您可以通过以下方式在Vibe和Spec模式之间切换：

  1. 在Chat界面中，点击左下角的模式切换按钮
  2. 选择"Vibe模式"或"Spec模式"
  3. 或者，在消息中明确指定模式，如"使用Spec模式生成代码"

示例对比

Vibe模式响应：

对于用户认证，您可以考虑多种方案。JWT令牌是一种流行选择，它无状态且易于扩展。另一种选择是Session-Cookie方案，它在服务器端存储状态，更传统但可能更安全。OAuth2.0则适合第三方登录场景。您的应用具体需求是什么？

Spec模式响应：

以下是JWT令牌认证的实现代码： ```javascript // auth.js const jwt = require('jsonwebtoken'); const generateToken = (user) => { return jwt.sign( { id: user.id, email: user.email }, process.env.JWT_SECRET, { expiresIn: '24h' } ); }; const verifyToken = (token) => { return jwt.verify(token, process.env.JWT_SECRET); }; module.exports = { generateToken, verifyToken }; ```

终端集成操作

KIRO IDE的Chat功能可以直接与终端集成，允许您通过自然语言执行命令行操作，这对于不熟悉命令行的开发者特别有用。

启用终端集成

要启用终端集成功能，请按照以下步骤操作：

  1. 在Chat界面中，点击右上角的设置图标
  2. 选择"集成设置"
  3. 启用"终端集成"选项
  4. 选择默认的终端类型（如bash、zsh、cmd等）

使用终端集成

启用后，您可以通过自然语言请求执行终端命令：

用户：创建一个新的React组件目录并生成基础文件 AI：我将执行以下命令： 1. 创建目录：mkdir src/components/UserProfile 2. 创建组件文件：touch src/components/UserProfile/UserProfile.js 3. 创建样式文件：touch src/components/UserProfile/UserProfile.css 4. 创建测试文件：touch src/components/UserProfile/UserProfile.test.js 是否执行？[是/否]

高级终端操作

您还可以执行更复杂的终端操作：

  • Git操作：提交代码、创建分支、合并更改
  • 包管理：安装依赖、更新包、运行脚本
  • 文件操作：搜索文件、批量重命名、权限修改
  • 系统管理：查看进程、监控资源、环境配置

安全提示

终端集成功能具有执行系统命令的能力，请谨慎使用。AI会显示将要执行的命令，并在执行前请求您的确认。避免执行不熟悉或可能造成系统损害的命令。

开发服务器管理

KIRO IDE的Chat功能可以帮助您管理开发服务器，包括启动、停止、配置和监控服务器状态。

服务器启动与配置

您可以通过Chat启动和配置开发服务器：

用户：为我的React应用启动开发服务器，并配置热重载和代理 AI：我将为您启动React开发服务器并配置热重载和代理。执行以下操作： 1. 启动开发服务器：npm start 2. 配置代理到API服务器：http://localhost:3001 3. 启用热重载模块替换 4. 设置浏览器自动打开 服务器已启动，访问地址：http://localhost:3000

服务器监控

Chat可以监控开发服务器的状态并提供实时反馈：

  • 性能监控：显示服务器资源使用情况
  • 错误追踪：捕获并报告服务器错误
  • 日志分析：分析服务器日志并提供见解
  • 请求分析：分析API请求模式和性能

多服务器管理

对于需要运行多个服务器的复杂项目，Chat可以帮助您：

  • 服务编排：按正确顺序启动多个服务器
  • 端口管理：自动分配和管理端口
  • 依赖处理：处理服务间的依赖关系
  • 统一控制：通过单一界面控制所有服务器

诊断工具使用

KIRO IDE提供了强大的诊断工具，可以帮助您识别和解决开发中的问题。

代码诊断

Chat可以分析您的代码并提供诊断信息：

  • 语法检查：识别语法错误和潜在问题
  • 性能分析：找出性能瓶颈和优化机会
  • 安全扫描：检测安全漏洞和风险
  • 代码质量：评估代码质量和可维护性

依赖诊断

Chat可以帮助您管理项目依赖：

  • 冲突检测：识别版本冲突和兼容性问题
  • 漏洞扫描：检查已知安全漏洞
  • 更新建议：推荐依赖更新和替代方案
  • 大小分析：分析依赖对打包大小的影响

运行时诊断

Chat可以诊断运行时问题：

  • 错误分析：解释错误消息和堆栈跟踪
  • 内存泄漏：检测内存泄漏和资源使用问题
  • 网络问题：诊断API请求和网络连接问题
  • 性能瓶颈：识别运行时性能问题

诊断示例

用户：我的React应用在加载大量数据时变得很慢 AI：经过分析，我发现了以下性能问题： 1. 组件未进行记忆化，导致不必要的重新渲染 2. 数据加载未实现分页或虚拟滚动 3. 图片资源未优化，加载时间过长 4. 状态更新过于频繁，未使用防抖 建议的解决方案： 1. 使用React.memo包装组件 2. 实现分页加载或react-window虚拟滚动 3. 添加图片懒加载和压缩 4. 对搜索输入使用useMemo和useCallback 是否需要我实现这些优化？

高级提示技巧

掌握高级提示技巧可以帮助您从Chat获得更准确、更有用的响应。

上下文提供

为AI提供充分的上下文是获得良好响应的关键：

  • 项目背景：描述项目类型、技术栈和目标
  • 代码上下文：提供相关代码片段或文件引用
  • 问题背景：解释问题的发生环境和影响
  • 约束条件：明确技术限制和业务要求

明确指令

使用明确、具体的指令：

  • 指定格式：要求特定代码格式或结构
  • 指定框架：明确使用的框架和版本
  • 指定风格：要求特定的代码风格或模式
  • 指定范围：明确任务的边界和范围

分步交互

对于复杂任务，使用分步交互：

  • 分解任务：将大任务分解为小步骤
  • 逐步确认：在每个步骤后请求确认
  • 迭代改进：基于反馈逐步改进结果
  • 中间检查：在关键点检查进度和方向

提示对比

低效提示：

帮我优化这个函数

高效提示：

我有一个React应用中的数据处理函数，它处理用户输入的搜索查询并过滤产品列表。当前实现在处理1000+产品时响应时间超过2秒。请帮我优化这个函数，要求： 1. 保持现有功能不变 2. 使用React的useMemo进行记忆化 3. 实现防抖功能，延迟300ms执行 4. 保持代码可读性和可维护性 5. 兼容React 18和TypeScript 以下是当前函数代码： [在此处粘贴代码]

反馈循环

建立有效的反馈循环：

  • 明确反馈：具体说明哪些部分好，哪些需要改进
  • 提供示例：给出期望结果的示例
  • 解释原因：解释为什么需要特定修改
  • 迭代优化：通过多轮对话逐步完善结果

最佳实践

以下是在使用KIRO Chat高级功能时的一些最佳实践：

项目组织

  • 为每个项目创建独立的Chat会话
  • 使用有意义的会话名称便于后续查找
  • 定期导出重要的Chat记录作为文档
  • 为常见任务创建提示模板

安全考虑

  • 避免在Chat中分享敏感信息（API密钥、密码等）
  • 使用环境变量存储敏感配置
  • 定期审查终端命令执行历史
  • 在生产环境前谨慎测试AI生成的代码

性能优化

  • 定期清理Chat历史以保持响应速度
  • 对于大型代码库，使用文件引用而非粘贴整个文件
  • 合理使用Autopilot，避免过度自动化
  • 监控AI响应时间，必要时调整上下文长度
